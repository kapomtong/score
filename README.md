# วิธี Deploy (key ซ่อนปลอดภัย ไม่ติด GitHub push protection)

## 1) หา API-Football Key เพิ่ม (ตัวใหม่ สำคัญที่สุด — ดึงนัดแข่ง+สถิติทีมจริง)

ตัวนี้คนละตัวกับ RapidAPI ที่เคยสมัครไว้ก่อนหน้า (ตัวเก่ายังไม่มีเอกสารยืนยัน endpoint ที่ใช้ได้แน่นอน)
API-Football มีเอกสารชัดเจนและฟรี 100 requests/วัน

1. ไปที่ dashboard.api-football.com → สมัครสมาชิก (ฟรี)
2. หลังยืนยันอีเมล เข้าไปที่หน้า Dashboard จะเห็น API Key ของคุณทันที (ไม่ต้องผูกบัตร)
3. เก็บ key นี้ไว้ตั้งเป็น `API_FOOTBALL_KEY` ในขั้นตอนที่ 4

## 2) อัปโหลดขึ้น GitHub
โฟลเดอร์นี้ไม่มี API key อยู่ในไฟล์ไหนเลย เพราะฉะนั้นอัปโหลดขึ้น GitHub ได้ปกติ
ไม่ถูก push protection บล็อกแน่นอน

1. สร้าง repo ใหม่บน github.com
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ (index.html, api/, package.json)

## 3) Import เข้า Vercel
1. ไปที่ vercel.com/new
2. เลือก "Import Git Repository" แล้วเลือก repo ที่เพิ่งสร้าง
3. ยังไม่ต้องกด Deploy — ไปตั้งค่า Environment Variables ก่อน (ข้อ 4)

## 4) ตั้งค่า Environment Variables (สำคัญที่สุด — นี่คือที่ซ่อน key)
ในหน้า Vercel ตอน import โปรเจกต์ (หรือทีหลังใน Project Settings → Environment Variables)
ให้เพิ่มตัวแปรทั้ง 4 นี้:

| Name                 | Value                              |
|----------------------|-------------------------------------|
| API_FOOTBALL_KEY     | (key จาก dashboard.api-football.com) |
| GOOGLE_SEARCH_KEY    | (Google Custom Search API Key)      |
| GOOGLE_SEARCH_CX     | (Search Engine ID / cx)             |
| GEMINI_KEY           | (Gemini API Key)                    |

ค่าพวกนี้จะถูกเก็บไว้ในระบบของ Vercel เท่านั้น ไม่ปรากฏในโค้ด ไม่ปรากฏใน GitHub
และผู้ใช้เว็บจะมองไม่เห็นเด็ดขาด

## 5) กด Deploy
หลังตั้งค่า Environment Variables ครบแล้ว กด Deploy
รอสักครู่จะได้ลิงก์เว็บ เช่น your-project.vercel.app

## วิธีใช้งานเว็บ
1. กด "โหลดนัดที่กำลังจะแข่ง" → ระบบดึงตารางฟุตบอลโลกจริงที่ยังไม่เริ่มแข่งมาแสดง
2. กด "เลือกวิเคราะห์นัดนี้" → ระบบดึงสถิติยิง/เสียประตูจริงของทั้งสองทีมมากรอกให้อัตโนมัติ
3. ปรับตัวเลขเองได้ถ้าต้องการ แล้วกด "คำนวณความน่าจะเป็น"
4. กด "ค้นหา" เพื่อดึงข่าวบาดเจ็บ/ฟอร์มทีมล่าสุดจริง
5. กด "ให้ Gemini อธิบายเหตุผล" เพื่อให้ AI สรุปเหตุผลจากข้อมูลทั้งหมด

## ข้อจำกัดที่ควรรู้
- สถิติทีมคำนวณจาก "เฉพาะนัดในทัวร์นาเมนต์นี้เท่านั้น" (ไม่รวมนัดกระชับมิตร/ฟุตบอลลีก) ถ้าทีมเพิ่งแข่งไป 2-3 นัด
  ตัวอย่างข้อมูลจะน้อย ความแม่นยำจึงมีจำกัด — ระบบจะเตือนไว้ในหน้าเว็บถ้าข้อมูลน้อยเกินไป
- Free tier ของ API-Football จำกัด 100 requests/วัน ถ้าใช้บ่อยเกินไปอาจโดน rate limit
- นี่คือเครื่องมือช่วยวิเคราะห์ตามความน่าจะเป็น ไม่ใช่การการันตีผลการแข่งขัน
